# Блокировка рекламы через пользовательские правила

- Karing по умолчанию интегрирует [ACL и geosite rules](https://github.com/KaringX/karing-ruleset/).
- Для блокировки рекламы можно напрямую использовать `acl:BanAD` и `geosite:category-ads`.

## Процесс настройки

### Шаги

#### 1. Включите правило

- Настройки -> Разделение трафика -> _Правила разделения_ -> кнопка редактирования справа вверху (значок ✏)
  - -> Пользовательская группа разделения -> кнопка ➕ справа вверху -> примечание `BanAD`
  - -> В списке правил выберите _BanAD_
  - -> Прокрутите до встроенных правил `Rule Set(build-in)`
  - Найдите и выберите `acl:BanAD` и `geosite:category-ads`
  - Нажмите √ справа вверху для сохранения

#### 2. Задайте действие правила

- Настройки -> Разделение трафика -> _Правила разделения_ -> созданная группа `BanAD` -> выберите **Блокировать**.

- Вернитесь на главную страницу и переподключитесь, чтобы правило вступило в силу.

### О category-ads

- geosite:category-ads содержит распространённые рекламные домены.
- geosite:category-ads-all содержит распространённые рекламные домены, а также домены рекламных провайдеров.

### Следующие шаги подходят для старых версий ниже v1.0.35.460

#### 1. Включите правило

- Настройки -> Разделение трафика -> _ACL_ -> включите `Включить`, войдите в ACL -> найдите и включите `BanAD`.
- Настройки -> Разделение трафика -> _GeoSite_ -> включите `Включить`, войдите в GeoSite -> найдите и включите `category-ads`.

#### 2. Задайте действие правила

- Настройки -> Разделение трафика -> _Правила разделения_ -> `ACL:BanAD` -> выберите **Блокировать**.
- Настройки -> Разделение трафика -> _Правила разделения_ -> `GeoSite:category-ads` -> выберите **Блокировать**.

- Вернитесь на главную страницу и переподключитесь, чтобы правило вступило в силу.
