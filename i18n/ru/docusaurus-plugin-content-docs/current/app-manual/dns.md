#### Включить правила разделения DNS: согласно настройкам [правил разделения](/app-manual/diversion-rule), соответствующий трафик использует соответствующие DNS-серверы для разрешения доменов

- [Прокси-сервер]: использовать DNS прокси-сервера для разрешения домена(в режиме новичка объединено с [прямым трафиком])
- [Прямой трафик]: использовать DNS прямого трафика для разрешения домена
- [Прокси-трафик]: использовать DNS прокси-трафика для разрешения домена(в режиме новичка объединено с [final])
- [final]: использовать final DNS для разрешения домена

#### [Прокси-трафик] разрешает DNS через прокси-сервер: прокси-трафик разрешает домены через прокси-сервер, а не напрямую подключается к DNS-серверу

- Зависит от уровня поддержки DNS прокси-сервером; некоторые прокси-серверы могут не поддерживать DNS или поддерживать его неполно
- DNS-разрешение соответствующего трафика полностью идет через прокси-канал, что безопаснее, но может быть медленнее

#### [final] разрешает DNS через прокси-сервер: final-трафик разрешает домены через прокси-сервер, а не напрямую подключается к DNS-серверу-

- Зависит от уровня поддержки DNS прокси-сервером; некоторые прокси-серверы могут не поддерживать DNS или поддерживать его неполно
- DNS-разрешение соответствующего трафика полностью идет через прокси-канал, что безопаснее, но может быть медленнее

#### Статический IP: пользовательское разрешение доменов, аналогично настройке файла hosts в Windows

#### Разрешать входящие домены: для трафика, приходящего из прокси, сначала выполнять разрешение домена, затем разделять по домену и IP. Подходит для случаев, когда в правилах разделения не настроены доменные правила

#### Тестовый домен: домен для разрешения при проверке задержки DNS

#### [TTL](/app-manual/glossary): время жизни/кеширования результата DNS-запроса

- Чем больше значение, тем дольше кеш и выше производительность; чем меньше значение, тем быстрее обновление
