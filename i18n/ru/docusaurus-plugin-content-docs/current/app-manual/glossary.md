#### Текущий выбор

- Автоматический выбор: автоматически выбирает узел с минимальной задержкой из списка кандидатов
- Прямое подключение: подключение напрямую, без прокси
- Пользовательская группа прокси/пользовательский автоматический выбор: пользовательский [автоматический выбор]
- Прокси-узел: конкретный прокси-сервер

#### DNS(служба доменных имен)

- Разделение: согласно правилам разделения использовать соответствующий DNS-сервер для разрешения домена
- ECS: EDNS Client Subnet
- FakeIP: технология для улучшения производительности/стабильности DNS-запросов в режиме TUN. DNS-запросы приложений через TUN перехватываются VPN и заменяются ответом с псевдо-IP. После последующего HTTP-запроса приложения запрос пересылается прокси-серверу; прокси-сервер выполняет DNS-разрешение домена, получает ответ и возвращает его запрашивающей стороне. Для трафика, которому не нужен прокси, запрос выполняется напрямую через VPN-сервер
- TTL: время жизни

#### Другое

- TUN: режим виртуальной сетевой карты, разновидность прозрачного прокси, не требует адаптации приложений(на разных платформах обработка немного отличается)
- Системный прокси: требует активной поддержки со стороны приложения(обходит TUN)
- Входящее подключение: подключение к VPN-сервису
- Исходящее подключение: подключение от VPN-сервиса наружу
- Потеря трафика: на главном экране Karing состояние подключения показывает, что подключение установлено, но у других приложений есть сетевые проблемы, например они постоянно находятся в состоянии обновления
- Разрыв подключения: на главном экране Karing состояние подключения показывает, что подключение разорвано. Возможные причины: фоновый VPN-процесс был завершен системой или вышел из-за ошибки
