# Базовая конфигурация провайдера base.json

## Описание

- `base.json` хранит базовую информацию провайдера. Он обновляется при смене состояния APP(например запуск, подключение, переход на передний план), минимальный интервал - 3 часа.

### Когда запрашивается соответствующая конфигурация провайдера?

- Когда пользователь входит к провайдеру через [заклинание](/cooperation/harry/spell) и завершает импорт конфигурации, Karing APP привязывается к провайдеру.
- В момент привязки начинается запрос соответствующей конфигурации провайдера(base.json), которая перезаписывает стандартные параметры Karing.

### Можно ли разместить самостоятельно

- Размещается только на сайте karing.

## Описание комментариев {#desc}

```jsx title="base.json"
{
    // base.json обновляется при смене vpn-состояния APP/переходе на передний план/запуске; минимальный интервал - 3 часа
    //
    // --- ---> ниже базовая информация провайдера
    // Название VPN-сервиса(провайдера)
    "name": "your service name",
    // Главная страница; китайские домены/русские домены/punycode автоматически преобразуются в ascii-домен
    "home": "https://your-homepage/",
    // Документация
    "faq": "https://your-homepage/docs",
    // Поддержка или тикеты
    "customer_service": "https://your-homepage/user/ticket",
    // Социальные сети
    "follow": "https://t.me/KaringApp",
    //
    // --- ---> ниже конфигурация APP
    // Адрес начального конфигурационного файла APP; по умолчанию размещается системой, может быть задан провайдером самостоятельно
    "config": "",
    // Адрес конфигурационного файла уведомлений APP; по умолчанию размещается системой, может быть задан провайдером самостоятельно
    "notice": "",
    // Интервал обновления уведомлений, единица: минуты. Используется для контроля интервала обновления notice. Минимум 10, по умолчанию 3 часа
    "notice_update_interval": 180,
    //
    // --- ---> ниже конфигурация интеграции с Karing
    // URL страницы интеграции; китайские домены/русские домены/punycode автоматически преобразуются в ascii-домен
    "connect": "",
    // Магические команды, используемые на странице привязки для перехода к VPN-сервису(провайдеру); один ISP может задать максимум 2 слова
    "spells": [
        '急速云',
        'RapidNetwork',
    ],
    //
    // --- ---> ниже генерируется системой
    // ProviderID назначается системой, уникальный идентификатор
    "pid": null,
    // При каждом изменении конфигурации система автоматически увеличивает значение на 1
    "version": 1
}

```
